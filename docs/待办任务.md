## 使用说明

每次只执行一个任务，完成后必须：

- 先阅读 `AGENTS.md`、`docs/architecture_notes.md`、`docs/硬件匹配分析.md`、`docs/pin_map.md`、`docs/项目结构.md` 和 `项目背景/` 下相关硬件资料。
- 如果已有文件和本任务描述重合，优先采用已有文件中的最新指示；本文件只是任务队列，不覆盖设计文档、硬件资料和代码事实。
- 如果任务需要真实硬件验证，而当前无法接触硬件，只能完成代码/文档准备，并把未验证项写入 `docs/需要补充的信息.md` 或进度记录。
- 保持 FreeRTOS 架构，不要把业务逻辑塞进 `loop()`；新代码按 `docs/architecture_notes.md` 放入对应分层。
- 保持真实硬件危险路径默认关闭，除非任务明确要求开启宏做编译验证；真实打印头、电机、VH 输出相关改动必须优先保证安全态。
- 更新 `docs/codex_progress.md` 和 `docs/项目结构.md`；如 `docx/项目结构.md` 仍作为结构快照存在，也同步更新。
- 能编译的任务至少运行 `python -m platformio run`；涉及真实硬件宏的任务额外运行对应宏开启编译。
- 每步完成后提交并推送。
