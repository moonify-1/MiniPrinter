## 使用说明

每次只执行一个任务，完成后必须：

- 先阅读 `AGENTS.md`、`docs/architecture_notes.md`、`docs/硬件匹配分析.md`、`docs/硬件补充信息整理.md`、`docs/pin_map.md`、`docs/项目结构.md` 和 `项目背景/` 下相关硬件资料。
- 如果已有文件和本任务描述重合，优先采用已有文件中的最新指示；本文件是任务队列，不覆盖设计文档、硬件资料和代码事实。
- 新增联网、API、打印文件、状态展示、控制命令时，优先走 WiFi API；不要把 UART 协议扩展成正式控制面。
- 保持 FreeRTOS 架构，不要把业务逻辑塞进 `loop()`；新代码按 `docs/architecture_notes.md` 放入对应分层。
- 保持真实硬件危险路径默认关闭，除非任务明确要求开启宏做编译验证；真实打印头、电机、VH 输出相关改动必须优先保证安全态。
- 更新 `docs/codex_progress.md` 和 `docs/项目结构.md`；如 `docx/项目结构.md` 仍作为结构快照存在，也同步更新。
- 能编译的任务至少运行 `python -m platformio run`；涉及真实硬件宏的任务额外运行对应宏开启编译。
- 每步完成后提交并推送。

## WiFi API 总原则

- WiFi 是正式通信方式：打印文件上传、启动打印、取消打印、走纸、状态查询、传感器查询、参数读写、错误清除、安全关闭、工厂测试都要有 API。
- 打印文件不能通过串口发送；正式路径必须是 WiFi 上传文件，再通过 WiFi API 启动打印任务。
- API 任务应放在 `services` / `protocol` / `tasks` 等合适层级：WiFi/HTTP 只负责收发请求，真正业务仍调用已有 service/app 层。
- 所有危险控制 API 必须先走安全检查：`safe-off` 优先级最高，`SAFE_MODE` 下只允许白名单 API。
- API 返回值使用结构化 JSON，至少包含 `ok`、`code`、`message`；涉及状态的 API 返回当前状态快照。
- 设备当前已知局域网 IP：`192.168.1.134`。后续可补 mDNS 或配置接口，但任务初期可先按固定 IP 调试。

## WiFi API 初始设计

建议统一使用 `/api/v1` 前缀。下面是后续任务应逐步实现的 API 清单。

| 类型 | 方法与路径 | 作用 | 对应现有能力 |
|---|---|---|---|
| 基础信息 | `GET /api/v1/info` | 返回固件版本、协议版本、功能开关、设备名 | `GET_INFO` |
| 运行状态 | `GET /api/v1/status` | 返回系统状态、打印状态、传感器摘要、电池摘要、错误摘要 | `GET_STATUS` |
| 健康状态 | `GET /api/v1/health` | 返回关键任务心跳、内存、运行时间、WDT 状态 | `HealthService` |
| 传感器 | `GET /api/v1/sensors` | 返回 paper、temp、battery、charge、motor_fault、valid | `SENSOR_TEST` |
| 电池 | `GET /api/v1/battery` | 返回 battery_mv、charge_status、low_power | `SensorService` / `ThermalSafety` |
| 当前错误 | `GET /api/v1/error` | 返回最近错误、严重级别、模块、详情、safe_mode | `GET_ERROR` |
| 清除错误 | `POST /api/v1/error/clear` | 清除可恢复错误 | `CLEAR_ERROR` |
| 日志 | `GET /api/v1/logs/recent` | 返回最近日志，便于网页调试 | `LogService` |
| 参数读取 | `GET /api/v1/params` | 返回当前参数快照 | `GET_PARAM` |
| 参数修改 | `PATCH /api/v1/params` | 修改允许开放的参数，必须范围校验 | `SET_PARAM` TODO |
| 参数保存 | `POST /api/v1/params/save` | 保存参数到 NVS | `SAVE_PARAM` |
| 恢复默认 | `POST /api/v1/params/factory-reset` | 恢复默认参数 | `FACTORY_RESET` |
| 安全关闭 | `POST /api/v1/safe-off` | 最高优先级关闭危险输出 | `SAFE_OFF` |
| 进入安全模式 | `POST /api/v1/safe-mode` | 主动进入 SAFE_MODE | `ENTER_SAFE_MODE` |
| 自检 | `POST /api/v1/self-test` | 执行安全自检 | `SELF_TEST` |
| 重启 | `POST /api/v1/reboot` | 安全关闭输出后重启 | `REBOOT` |
| 上传打印文件 | `POST /api/v1/print/files` | 创建打印文件上传会话，返回 `file_id` | 新增 |
| 上传分片 | `PUT /api/v1/print/files/{file_id}/chunks/{index}` | 上传打印数据分片，支持断点和校验 | 新增 |
| 完成上传 | `POST /api/v1/print/files/{file_id}/complete` | 校验总大小、行数、CRC/SHA 后入队准备 | 新增 |
| 文件列表 | `GET /api/v1/print/files` | 查看已上传但未清理的文件 | 新增 |
| 删除文件 | `DELETE /api/v1/print/files/{file_id}` | 删除上传文件 | 新增 |
| 启动打印 | `POST /api/v1/print/jobs` | 以 `file_id` 创建并启动打印任务 | `PRINT_START/LINE/END` 的 WiFi 替代 |
| 当前打印 | `GET /api/v1/print/jobs/current` | 查看当前任务进度、行号、状态 | `PrintService` / `PrintSpooler` |
| 取消打印 | `POST /api/v1/print/jobs/current/cancel` | 取消当前打印任务 | `PRINT_CANCEL` |
| 走纸 | `POST /api/v1/feed` | 安全检查后走纸 | `FEED` |
| 电机测试 | `POST /api/v1/factory/motor-test` | 低速小步数电机测试 | `MOTOR_TEST` |
| 打印头移位测试 | `POST /api/v1/factory/head-shift-test` | 只 shift/latch，不打开 VH | `HEAD_SHIFT_TEST` |
| 打印头 STB 测试 | `POST /api/v1/factory/head-stb-test` | VH 关闭条件下测 STB 脉冲 | `HEAD_STB_TEST` |

打印文件建议先支持 384 点宽、48 字节/行的 raw 数据。后续如需图片格式转换，应在 API 层外或单独工具中处理，固件内部先保持简单。
