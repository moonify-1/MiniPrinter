# MiniPrinter 电池与充电状态策略

更新时间：2026-04-26 22:03:34

## 1. 当前硬件路径

- `EQD`：2S 电池电压分压输入，按 `220k/100k` 先估算为 `ADC_mV * 3.2`。
- `BAT_STAT`：IP2326 充电状态输出，语义尚未确认，固件暂时返回 `UNKNOWN`。
- `G_NFAULT`：DRV8833 低有效故障输入，故障时禁止继续电机/打印相关动作。

## 2. 低电压保护线

当前保守阈值为：

```text
SAFETY_DEFAULT_LOW_BATTERY_STOP_MV = 6500mV
```

策略：

- `battery_mv >= 6500` 且读数有效：设置 `EVT_BAT_OK`。
- `battery_mv < 6500` 且读数有效：清除 `EVT_BAT_OK`，设置 `EVT_LOW_POWER`。
- 读数无效：清除 `EVT_BAT_OK` 和 `EVT_LOW_POWER`，危险动作仍会因为传感器无效或电池不足被拒绝。

## 3. WiFi API 展示

- `GET /api/v1/status` 的 `sensors` 中返回 `battery_mv`、`charge_status`、`charge_status_raw`、`charging`。
- `GET /api/v1/battery` 返回 `battery_mv`、`battery_ok`、`low_power`、`low_voltage_stop_mv`、`charge_status`。

## 4. 危险动作限制

- 热敏头加热路径继续通过 `ThermalSafety_CheckCanPrint()` 检查低电压。
- 真实步进电机工厂测试在 `MP_ENABLE_HW_STEPPER=1` 时也会检查低电压和 `motorFault`。
- 固件不控制 IP2326 充电过程，只读取状态并用于显示/保护。

## 5. 待用户实测

- `BAT_STAT` 高低电平与未充电、充电中、充满、故障之间的对应关系。
- `EQD` ADC 原始值和万用表电池电压的对应关系。
- 真实打印瞬间 VH_OUT 是否存在明显压降。
