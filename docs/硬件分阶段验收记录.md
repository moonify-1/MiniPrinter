# MiniPrinter 真实硬件分阶段验收记录

更新时间：2026-04-26 22:16:38

## 1. 总原则

- Codex 只完成代码、脚本、文档和记录模板；真实电平、波形、电机方向、首次加热打印必须由用户在实物上填写。
- 任一阶段失败，必须停止后续更危险阶段，先执行 `POST /api/v1/safe-off`，必要时断电。
- 正式验收优先使用 WiFi API；UART 只作为底层临时调试参考。
- 未完成空载波形和 safe-off 验证前，不得连接或启用真实加热打印。

## 3. 失败处理

| 失败类型 | 处理 |
|---|---|
| API 无响应 | 检查 IP、WiFi 模式、串口启动日志；不要进入硬件动作阶段。 |
| `safe-off` 不成功 | 立即断电，检查 `Bsp_SetAllOutputsSafe()` 和 GPIO 映射。 |
| `G_VH` 未保持关闭 | 停止所有热敏头测试，复核 MOS 控制方向和 `PIN_G_VH`。 |
| STB 波形异常 | 停止真实打印，复核 STB GPIO、FPC 和驱动时序。 |
| 电机方向错误 | 不进入真实打印，先修正相序或方向策略。 |
| 电池电压异常或低电压 | 禁止加热和电机危险动作，复核 `EQD` ADC 与万用表读数。 |
| 首次真实打印异常 | 立即 safe-off 或断电，记录图案、脉宽、温度、电压和现象。 |
