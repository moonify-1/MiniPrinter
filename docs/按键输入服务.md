# MiniPrinter 按键输入服务

更新时间：2026-04-26 21:58:47

## 1. 定位

`G_KEY` 已接入固件状态读取，但第一阶段只用于观察短按/长按事件，不直接触发打印、走纸、工厂测试或其它危险动作。正式控制面仍是 WiFi API。

## 2. 当前实现

- `src/services/key_service.*` 负责按键输入、去抖、短按/长按识别和快照保存。
- `TaskSensor` 每 50ms 调用一次 `KeyService_Poll()`。
- `GET /api/v1/status` 返回 `key` 字段。
- `GET /api/v1/key` 返回独立按键快照。

## 3. 当前假设

- 当前按 `INPUT_PULLUP` 和低有效处理：`raw_level_high=false` 表示按下。
- 该有效电平尚未实测确认，因此 API 返回 `active_low_assumed=true`。
- 短按/长按仅作为事件展示，不会自动触发危险动作。

## 4. 后续实测

| 项目 | 需要用户确认 |
|---|---|
| 空闲电平 | 松开按键时 `G_KEY` 是高电平还是低电平。 |
| 按下电平 | 按下时 `G_KEY` 是否稳定变为低电平。 |
| 抖动情况 | 40ms 去抖是否足够，是否需要调整。 |
| 交互含义 | 后续是否把短按/长按绑定到安全动作或普通 UI 动作。 |
