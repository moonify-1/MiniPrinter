# 当前项目实现的打印机功能

更新时间：2026-04-26

本文只描述当前源码已经实现的能力，不把“计划中功能”写成“已完成”。当前结论是：项目已经具备 FreeRTOS 下的小型热敏打印机软件闭环和安全骨架；默认运行仍以 mock 驱动为主，真实热敏头、真实步进电机路径需要显式开启宏，并且还缺少真实硬件验证。

## 1. 已实现的整机基础能力

- 启动安全态：`setup()` 最早调用 `Bsp_PreInitSafeOutputs()`，随后 `Bsp_Init()` 再次收敛输出，默认关闭 VH、STB、打印移位信号和 DRV8833 驱动输入。
- FreeRTOS 对象：已经创建系统事件组、日志队列、错误队列、命令队列、打印控制队列、行缓冲队列、传感器队列和互斥量。
- FreeRTOS 任务：已经有日志、系统、传感器、参数、打印引擎、命令、协议、监控任务。
- 系统状态机：支持 `BOOT -> INIT -> SELF_TEST -> IDLE/READY/RUNNING`，也支持 `ERROR`、`RECOVERY`、`SAFE_MODE`、`LOW_POWER` 等状态。
- 任务健康监控：关键任务通过 heartbeat 上报状态，监控任务发现关键任务超时后会关闭危险输出并上报错误。
- 可选 Watchdog：`MP_ENABLE_WDT=1` 时可接入 ESP32 Task Watchdog，默认关闭。

## 2. 已实现的通信和命令能力

- UART 协议帧：支持固定 magic、版本号、头长度、设备号、序号、命令号、payload 长度和 CRC16 校验。
- 串口接收任务：`ProtocolTask` 从 `Serial` 轮询读取字节，逐字节喂给协议解析器，完整帧投递到命令队列。
- 命令分发任务：`CommandTask` 从队列取出完整帧，并交给 `ProtocolService` 处理。
- 基础命令：支持 `PING`、`GET_INFO`、`GET_STATUS`、`GET_ERROR`、`CLEAR_ERROR`、`SELF_TEST`、`REBOOT`。
- 参数命令：支持 `GET_PARAM`、`SAVE_PARAM`、`FACTORY_RESET`；`SET_PARAM` 当前仍返回 TODO。
- 打印命令：支持 `PRINT_START`、`PRINT_LINE`、`PRINT_END`、`PRINT_CANCEL`、`FEED`。
- 工厂测试命令：支持 `SAFE_OFF`、`SENSOR_TEST`、`MOTOR_TEST`、`HEAD_SHIFT_TEST`、`HEAD_STB_TEST`、`ENTER_SAFE_MODE`。
- SAFE_MODE 白名单：进入安全模式后，只允许状态、错误、自检、重启、恢复默认、安全关闭等安全命令；打印类命令会被拒绝。

## 3. 已实现的打印数据接收与缓存

- 打印宽度：按 384 点/行、48 字节/行、单色 1 bit/点处理。
- STB 分组：一行拆成 6 个 STB 组，每组 64 点。
- 固定行缓冲池：使用 64 个 `LineBuffer`，不在打印接收路径动态分配内存。
- 空闲/就绪队列：`qLineFree` 保存空闲行缓冲，`qLineReady` 保存等待打印的行缓冲指针。
- `PRINT_LINE` 入队：payload 必须正好 48 字节；入队时会记录 jobId、lineNo，并统计黑点数量。
- 作业清理：取消或异常时会清空还没打印的行，并把行缓冲归还到固定池。

## 4. 已实现的打印执行链路

- 打印作业状态：`PrintApp` 记录打印状态、jobId、已打印行数、最后错误和结束请求。
- 打印控制队列：`PRINT_START`、`PRINT_END`、`PRINT_CANCEL`、`FEED` 都转换成 `PrintControlMsg` 交给 `PrintEngineTask`。
- 打印活动事件：开始打印时设置 `EVT_PRINT_ACTIVE`，结束、取消或失败后清除，系统状态可进入或退出 `RUNNING`。
- 行打印计划：每行会按 6 个 STB 组统计黑点数量，再计算每组热脉冲宽度。
- mock 打印路径：默认情况下使用热敏头 mock 和步进电机 mock，可以跑通“接收行 -> 安全检查 -> 模拟 shift/latch/STB -> 模拟走纸 -> 释放行缓冲”的软件流程。
- 真实打印保护路径：同时开启 `MP_ENABLE_HW_THERMAL_HEAD=1` 和 `MP_ENABLE_HW_STEPPER=1` 后，才允许进入真实硬件路径。
- 真实行流程骨架：真实路径按 `shiftLine48Bytes -> latch -> setVh(true) -> 逐组 STB pulse -> allStbOff -> setVh(false) -> stepper 走纸 -> 释放行缓冲` 执行。
- 异常收敛：任意打印步骤失败时，会关闭 STB/VH、释放电机、执行 `Bsp_SetAllOutputsSafe()`、清空行队列并上报错误。

## 5. 已实现的热敏打印头能力

- 热敏头统一接口：定义了 `ThermalHeadDriver`，上层不直接操作 GPIO/SPI。
- mock 热敏头：默认路径不操作真实 GPIO，只用于验证流程。
- ESP32 热敏头骨架：开启 `MP_ENABLE_HW_THERMAL_HEAD=1` 时编译真实 SPI/GPIO 路径。
- 48 字节移位：真实路径使用 SPI 向打印头发送一行 48 字节数据。
- LAT 锁存：提供 `latch()`，当前用 GPIO 翻转和微秒级延时实现。
- VH 控制：打开 VH 前会确认所有 STB 处于 inactive，避免意外形成加热通路。
- STB 分组脉冲：支持 6 个 STB 组逐组 pulse，并在 driver 层再次限制脉冲上限。
- 打印头调试测试：`HEAD_SHIFT_TEST` 只验证 shift/latch，不开 VH；`HEAD_STB_TEST` 默认也保持 VH 关闭，只用于观察 STB 波形。

## 6. 已实现的走纸/电机能力

- 步进电机统一接口：定义了 `StepperDriver`，上层通过接口调用，不直接写 DRV8833 GPIO。
- mock 电机：默认路径可模拟 wake、step、release、sleep 等动作。
- DRV8833 骨架：开启 `MP_ENABLE_HW_STEPPER=1` 时编译真实 DRV8833 GPIO 路径。
- 安全初始化：真实驱动 `init()` 首先让 DRV8833 休眠，并关闭 H 桥输入。
- 低速全步相序：实现了清晰的 4-step 全步相序，便于初期用万用表或示波器检查。
- fault 保护：按 nFAULT 低电平有效处理，检测到 fault 后立即 release 并 sleep。
- 走纸命令：`FEED` 当前走 mock 路径，并限制最大步数，避免一条命令触发长时间运动。

## 7. 已实现的传感器与安全判断

- 传感器统一接口：定义纸张、打印头温度、电池电压、充电状态、电机故障读取接口。
- mock 传感器：当前默认返回有纸、25℃、7400mV、未知充电状态、无电机故障。
- 传感器任务：周期性采样 mock 传感器，并发布最新 `SensorSnapshot`。
- 系统事件更新：根据纸张、温度、电池和电机故障更新 `EVT_PAPER_PRESENT`、`EVT_TEMP_OK`、`EVT_BAT_OK`、`EVT_DRV_READY`。
- 温度有效性判断：温度低于 -20℃ 或高于 120℃ 会被视为无效。
- 电池有效性判断：电池电压低于 3000mV 或高于 9000mV 会被视为无效。
- 打印许可判断：缺纸、温度无效、过温、低电压、电机故障都会禁止继续加热。
- 热脉冲计算：根据黑点数量、温度和电压计算保守脉冲宽度，并受参数和硬上限共同限制。
- 单组黑点限制：单个 STB 组同时加热点数超过安全上限时拒绝打印。

## 8. 已实现的错误和安全模式能力

- 统一错误码：错误码包含严重级别、恢复建议、模块号和模块内错误号。
- 错误事件：`ErrorEvent` 记录 code、severity、module、timestamp、detail。
- 错误上报服务：`Error_Report()` 会保存最后错误、投递错误队列、设置系统事件并写日志。
- SAFE_MODE 锁存：FATAL 或要求进入安全模式的错误会锁存安全模式请求。
- 系统错误处理：`SystemTask` 根据 WARN、ERROR、FATAL 做不同处理，FATAL 会进入 `SAFE_MODE`。
- 最高优先级安全关闭：`SAFE_OFF` 无论当前状态如何都会立即执行危险输出关闭。

## 9. 已实现的参数和持久化能力

- 默认参数块：包含打印、电机、传感器、通信、日志、安全参数。
- 参数 CRC32：保存前会计算 CRC32，用于判断 NVS 参数是否损坏。
- NVS 读取：启动时尝试从 Arduino Preferences/NVS 读取参数。
- 默认回退：NVS 不可用、参数缺失、版本不匹配、长度不匹配或 CRC 错误时，回退默认参数。
- 延迟保存：参数保存请求进入 `qParam`，由低优先级参数任务延迟合并写入 Flash。
- 恢复默认：`FACTORY_RESET` 会恢复默认参数并请求保存。

## 10. 已实现的主机侧辅助能力

- `tools/send_frame.py`：可构造并发送 `ping`、`status`、`safe-off`、`sensor-test`、`print-test` 协议帧。
- `tools/make_test_print.py`：可生成 384 点宽、48 字节/行的测试打印数据。
- dry-run 支持：主机脚本可在不打开串口时打印帧内容，便于先检查协议格式。

## 11. 当前还不能算完成的功能

- 还不能确认“已经真实打印出纸面内容”：目前没有真实串口、GPIO 波形、打印头加热、电机走纸的验证记录。
- 默认不会驱动真实打印头和真实电机：`MP_ENABLE_HW_THERMAL_HEAD` 与 `MP_ENABLE_HW_STEPPER` 默认都是 0。
- 当前所有 BSP 引脚仍是 `PIN_UNASSIGNED`：真实 GPIO 映射还没有落入 `src/bsp/bsp_pins.h`。
- 真实传感器驱动还没接入：纸张、温度、电池、电机 fault 当前仍走 mock。
- `SET_PARAM` 还没完成：能读参数、保存参数、恢复默认，但还不能通过协议正式修改单个参数。
- 自动化测试还不足：`test/` 目录目前只有 README，协议解析、行缓冲边界、安全模式白名单、热敏安全计算还缺少单元测试。

## 12. 一句话结论

当前项目已经实现了“打印机固件应有的软件组织、协议接收、行缓存、打印调度、安全检查、mock 打印闭环、真实硬件安全骨架和调试命令”；但距离“确认可以真实稳定打印”还差 GPIO 映射、真实传感器接入、硬件宏开启后的分阶段波形验证和实物打印验证。
