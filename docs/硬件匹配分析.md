# 硬件匹配分析

更新时间：2026-04-27

本文基于 `项目背景/` 目录下的原理图、BOM、ESP32-S3 DevKitC-1 GPIO 映射图和关键器件规格书，对照当前固件实现，判断硬件是否匹配、哪些功能还没有匹配到代码或实测记录。

## 1. 当前结论

当前文档已更新到最新项目状态。旧版本中“`src/bsp/bsp_pins.h` 仍全部为 `PIN_UNASSIGNED`、缺少 `EQD`、没有按键服务、真实传感器仍完全 mock”等判断已经过时。

当前固件与硬件方向总体匹配：硬件确实是一套 `ESP32-S3-WROOM-1-N16R8 + JX-2R-01 热敏机芯 + DRV8833 步进电机驱动 + IP2326 2S 充电管理 + TPS563200 3.3V Buck` 的小型热敏打印机板；固件已经围绕这套硬件建立了 GPIO 映射、安全输出、WiFi API、打印文件上传、mock 打印闭环、真实热敏头/步进电机驱动骨架、真实传感器第一阶段骨架和按键服务。

但当前仍不能宣称“已经适配真实硬件并可稳定打印”。原因是缺少真实设备逐阶段验收记录：WiFi 联机、安全态、GPIO 空载波形、VH/STB 关闭验证、电机方向、`PAPER_N/TM1/BAT_STAT` 实测、低密度短行真实打印都还需要用户实测。

## 3. 主要硬件模块与当前固件匹配情况

### 3.1 打印机芯 FPC

JX-2R-01 机芯规格书给出 384 点/行、48 字节/行、6 个 STB 组、每组 64 点、8 dots/mm、步进距离 0.0625mm。当前固件的打印几何参数与这些值一致。

| 机芯信号 | 规格书接口 | 原理图连接 | 当前固件状态 |
|---|---|---|---|
| `VSEN` | Pin 2 | `BUCK_OUT` 经 `R21=180R` | 硬件存在，固件未单独建模 |
| `PHE` | Pin 3 | 进入 LM393 缺纸检测电路，输出 `PAPER_N` | GPIO 已分配；真实驱动暂按始终有纸处理 |
| `VH` | Pin 6/7/25/26 | `G_VH` 控制高边 P-MOS 输出 `VH_OUT` | `PIN_G_VH=GPIO8`；高通低断；safe-off 默认关闭 |
| `DI` | Pin 8 | 经 22R 接 MCU | `PIN_DI=GPIO9`；真实热敏头宏开启后走 SPI 写入 |
| `CLK` | Pin 9 | 经 22R 接 MCU | `PIN_CLK=GPIO10`；由 SPI 时钟驱动 |
| `GND` | Pin 10/11/21/22 | 接地 | 匹配 |
| `STB1~STB6` | Pin 20/19/18/14/13/12 | 经 22R 接 MCU | `GPIO40/39/38/37/36/35`；默认低电平关闭 |
| `Vdd` | Pin 15 | `BUCK_OUT`，约 3.3V | 代码按 3.3V 逻辑时序设计 |
| `TM` | Pin 16/17 | `TM1` 热敏电阻采样网络 | `PIN_TM1=GPIO2`；ADC 已分配，温度换算待校准 |
| `LAT` | Pin 23 | 经 22R 接 MCU | `PIN_LAT=GPIO3`；真实热敏头宏开启后输出 latch 脉冲 |
| `DO` | Pin 24 | 未连接 | 固件未使用，单机芯非级联场景可接受 |
| 电机 A/B 相 | Pin 27~30 | 接 DRV8833 `AOUT/BOUT` | 有 DRV8833 4-step 全步驱动骨架，方向待实测 |

说明：原理图里 FPC 符号显示了额外机械/固定脚位，BOM 中连接器为 `X10A20H30G` 30P；最终装配仍应以实物 FPC 插入方向和连通性复核为准。

### 3.2 MCU 网络到 GPIO 映射

网络到 GPIO 已写入 `docs/pin_map.md` 和 `src/bsp/bsp_pins.h`，并且用户已用万用表完成连通性复核。下面这些映射已经是当前固件常量，不再是推断占位。

| 原理图网络 | GPIO | 当前代码状态 | 仍需验证 |
|---|---:|---|---|
| `PAPER_N` | GPIO5 | 已分配输入；真实传感器驱动暂固定有纸 | 有纸/无纸实际电平 |
| `G_NFAULT` | GPIO6 | 已分配输入；低有效故障 | 电机测试中的 fault 行为 |
| `G_NSLEEP` | GPIO7 | 已分配输出；低电平休眠 | safe-off 后保持低 |
| `G_KEY` | GPIO15 | 已接入按键服务，默认内部上拉、低有效假设 | 按下/松开实际电平 |
| `BAT_STAT` | GPIO16 | 已分配输入；当前返回 `UNKNOWN` | IP2326 状态语义和实测电平 |
| `EQD` | GPIO17 | 已分配 ADC；按 220k/100k 约 3.2 倍换算 | ADC 校准、万用表比对 |
| `G_VH` | GPIO8 | 已分配输出；高通低断，低电平关闭 VH | safe-off 和空载波形 |
| `LAT` | GPIO3 | 已分配输出 | latch 脉宽和时序 |
| `DI` | GPIO9 | 已分配输出 | SPI 数据波形 |
| `CLK` | GPIO10 | 已分配输出 | SPI 时钟波形 |
| `AIN1/AIN2` | GPIO11/GPIO12 | 已分配 DRV8833 A 相输入 | 电机方向和相序 |
| `BIN1/BIN2` | GPIO13/GPIO14 | 已分配 DRV8833 B 相输入 | 电机方向和相序 |
| `TM1` | GPIO2 | 已分配 ADC 输入 | NTC ADC 原始值和温度曲线 |
| `STB1~STB6` | GPIO40/39/38/37/36/35 | 已分配输出；高有效，默认低 | 空载 STB 脉冲和 VH 关闭 |

GPIO35~40 可作为普通 GPIO 输出、GPIO38/RGB LED 风险已由用户确认不阻塞；但真实热敏头启用前仍必须用示波器或逻辑分析仪确认波形。

### 3.3 电源与功率路径

- `IP2326`：负责 Type-C 输入、2S 锂电池升压充电和 `BAT_STAT` 状态输出。固件目前只读取/展示状态，不控制充电过程。
- `TPS563200DDCR`：通过反馈电阻生成约 3.3V `BUCK_OUT`，为 ESP32/打印头逻辑供电。
- `VH_OUT`：由 `G_VH -> N-MOS -> P-MOS` 控制，把 `2S+` 切到打印头 VH。用户已确认 `G_VH` 高通低断。
- `EQD`：`2S+ -> 220k/100k 分压 -> 1k/100nF 滤波 -> MCU ADC`，固件第一阶段按 3.2 倍换算电池电压。
- `H3`：只作为烧录/调试时的供电隔离跳帽，不作为固件运行期控制对象。

调试时可以断开底板给开发板的正电源，让 USB 给 ESP32 供电、底板电池给外设供电；前提是开发板和底板 GND 共地，并且不要在 USB 插着时合上供电跳帽。

### 3.4 缺纸、温度、电池、按键

- 缺纸：硬件有 LM393 和 `PAPER_N`，规格书倾向有反射低电平、无反射高电平。当前固件第一阶段固定返回有纸，避免未实测前误触发保护。
- 温度：硬件使用 `TM1` 和 30k/B3950 NTC。当前固件保守返回 25℃占位，等待室温和升温 ADC 原始值后再做 NTC 换算。
- 电池：`EQD` 已接入真实传感器骨架，按分压比换算 `batteryMv`；最终仍需 ADC 校准。
- 充电状态：`BAT_STAT` 已分配输入，但当前返回 `UNKNOWN`；IP2326 数据手册说明涓流低电平、退出涓流后高电平，但一根状态脚能否区分充满/异常仍需确认。
- 按键：`G_KEY` 已接入 `KeyService`，支持 40ms 去抖、短按/长按状态展示；当前不直接触发打印、走纸或危险动作。

## 4. 已匹配的功能

- 打印几何：384 点/行、48 字节/行、6 个 STB 组、每组 64 点。
- 同时加热点：限制单组不超过 64 dots，符合规格书同时打印点数要求。
- 走纸几何：默认 `stepsPerPrintLine=2`，对应 8 dots/mm 的 0.125mm 行距和 0.0625mm/step。
- VH 安全：safe-off 和热敏头驱动都会关闭 VH/STB；打开 VH 前检查 STB inactive。
- STB 方向：高电平有效，默认低电平关闭。
- DRV8833 方向：`nSLEEP` 低电平休眠，`nFAULT` 低有效；真实驱动检测 fault 后 release/sleep。
- 电机限流硬件：原理图 `R15/R18=560mR`，与机芯 0.357A 相电流方向匹配。
- GPIO 映射：所有核心网络已写入 `src/bsp/bsp_pins.h`，并在 `docs/pin_map.md` 记录。
- 正式控制面：WiFi API 已覆盖状态、传感器、电池、参数、打印文件上传、启动打印、取消、走纸和工厂测试。
- 主机工具：`tools/api_client.py` 用于 WiFi API；`tools/make_test_print.py` 用于生成 384 点宽 raw 文件；`tools/send_frame.py` 仅作 UART 底层调试。

## 5. 未完全匹配或仍需实测的功能

### 5.1 真实硬件验收尚未完成

当前最关键缺口不是“没有代码”，而是还没有真实设备验收记录。必须按 `docs/硬件分阶段验收记录.md` 完成：

- WiFi API 联机。
- 上电默认安全态。
- `safe-off` 后 `G_VH/STB/G_NSLEEP/AIN/BIN` 保持安全。
- `head-shift-test` 的 DI/CLK/LAT 波形。
- VH 关闭条件下 `head-stb-test` 的 STB 波形。
- `motor-test --steps 1` 的真实走纸方向。
- 低密度短行真实打印。

### 5.2 `PAPER_N` 未接入真实判断

GPIO 已分配，但当前真实传感器驱动仍固定返回有纸。后续需要实测有纸/无纸电平，再决定是否按 `PAPER_N == LOW` 转成 `paperPresent=true`。

### 5.3 `TM1` 温度保护未校准

GPIO 已分配，硬件 NTC 参数已知，但当前没有 ADC 原始值校准。真实过温保护不能只依赖 25℃占位值。

### 5.4 `BAT_STAT` 状态语义未闭环

IP2326 文档可确认部分电平语义，但固件目前仍返回 `UNKNOWN`。后续需要结合实际充电、满电、异常状态测量，确认是否能可靠映射到 `NOT_CHARGING/CHARGING/FULL/FAULT`。

### 5.5 电机方向和相序未实测

DRV8833 驱动已有 4-step 全步表，默认每打印行走 2 步。但实际正向走纸方向取决于 `AOUT/BOUT` 接线和机芯安装，必须低速小步数实测后确认。

### 5.6 打印头热模型仍是保守骨架

规格书给出 7.2V、64 点同时加热、25℃ 典型 Ton 约 0.49ms。当前代码已有最大点数、温度、电压、脉宽上限等安全限制，但打印浓度、温升、冷却和行周期仍需实测校准。

### 5.7 WiFi API 尚未做产品级配网和鉴权

当前 WiFi API 适合实验室和局域网调试，还没有鉴权、HTTPS、mDNS、配网 UI 和长期稳定性验证。正式产品化前需要单独设计。

### 5.8 固件侧自动化测试不足

`test/` 目录仍缺少协议解析、打印缓存边界、SAFE_MODE 白名单、参数边界和热安全计算等单元测试。

## 6. 推荐后续顺序

1. 先按 `docs/项目启动说明.md` 烧录 `MP_ENABLE_WIFI=1` 固件，完成 WiFi API 联机。
2. 执行 `safe-off`，用万用表/示波器确认所有危险输出安全态。
3. 采集 `PAPER_N` 有纸/无纸电平、`TM1` ADC 原始值、`BAT_STAT` 状态电平。
4. 开启 `MP_ENABLE_HW_SENSORS=1`，验证 `EQD` 电池电压和 `G_NFAULT`。
5. 开启 `MP_ENABLE_HW_THERMAL_HEAD=1`，在 VH 关闭条件下验证 shift/latch/STB 波形。
6. 开启 `MP_ENABLE_HW_STEPPER=1`，只做小步数低速电机测试，确认方向和 release/sleep。
7. 最后才开启真实短行低密度打印，随时准备 `safe-off` 或断电。

## 7. 当前判断

- 硬件设计与固件架构方向：匹配。
- GPIO 映射、WiFi API、上传/启动打印、按键、电池 ADC 骨架、真实热敏头/电机骨架：已落地。
- `PAPER_N`、`TM1`、`BAT_STAT`、电机方向、热模型、真实打印效果：仍需实测和校准。
- 当前项目可用于软件闭环和分阶段硬件验收，但不能跳过验收直接进行真实加热打印。
