# 项目启动说明

更新时间：2026-04-27

本文说明当前项目应该怎样启动、怎样传输打印文件、怎样调用打印机开始打印。当前正式控制面是 WiFi API；UART/串口只作为烧录和临时调试通道，不作为正式打印文件传输入口。

## 1. 当前使用方式总览

当前固件有两种使用状态：

| 使用状态 | 作用 | 是否会真实加热/走纸 |
|---|---|---|
| 默认安全状态 | 验证 RTOS、WiFi API、上传、打印队列和 mock 打印流程 | 不会 |
| 真实硬件状态 | 分阶段验证传感器、电机、打印头波形和低密度短行打印 | 会，必须谨慎 |

默认情况下这些危险硬件宏都是关闭的：

- `MP_ENABLE_HW_THERMAL_HEAD=0`
- `MP_ENABLE_HW_STEPPER=0`
- `MP_ENABLE_HW_SENSORS=0`
- `MP_ENABLE_WIFI=0`

所以想用 WiFi API 操作设备，至少要用 `MP_ENABLE_WIFI=1` 编译烧录。

## 2. 推荐启动流程

### 2.1 编译并烧录 WiFi API 固件

如果先用 AP fallback，不配置路由器账号：

```powershell
$env:PLATFORMIO_BUILD_FLAGS='-DMP_ENABLE_WIFI=1'
python -m platformio run -t upload
Remove-Item Env:PLATFORMIO_BUILD_FLAGS
```

固件会启动名为 `MiniPrinterRTOS` 的 AP。AP 模式下 ESP32 默认网关通常是 `192.168.4.1`，如果访问失败，以串口启动日志或电脑网络信息为准。

如果要连接已有局域网：

```powershell
$env:PLATFORMIO_BUILD_FLAGS='-DMP_ENABLE_WIFI=1 -DMP_WIFI_SSID=\"你的WiFi名\" -DMP_WIFI_PASSWORD=\"你的WiFi密码\"'
python -m platformio run -t upload
Remove-Item Env:PLATFORMIO_BUILD_FLAGS
```

当前文档中记录过的开发设备 IP 是 `192.168.1.134`。如果路由器重新分配了地址，用串口日志、路由器后台或网络扫描确认实际 IP。

### 2.2 上电后的第一组检查

把下面命令里的 `<ip>` 换成设备 IP：

```powershell
python tools/api_client.py --base http://<ip> info
python tools/api_client.py --base http://<ip> status
python tools/api_client.py --base http://<ip> safe-off
python tools/api_client.py --base http://<ip> sensors
python tools/api_client.py --base http://<ip> battery
```

判断标准：

- `info` 返回 `ok=true`，并且 `wifi_control_surface=official`。
- `status` 能看到系统状态、打印状态、传感器摘要和错误摘要。
- `safe-off` 返回 `ok=true`，危险输出保持关闭。
- `sensors` 和 `battery` 能返回读数；真实传感器未校准前，`PAPER_N`、`TM1`、`BAT_STAT` 可能仍是保守占位。

## 3. 如何传输打印文件

### 3.1 当前文件格式

第一阶段只支持 raw 单色位图：

- 每行宽度：384 点。
- 每行字节：48 bytes。
- 文件大小必须是 48 的整数倍。
- 当前最大文件：3072 bytes，也就是 64 行。
- 上传时会校验 CRC32。

固件不负责把 PNG/JPG/BMP 转成打印 raw。现在先用 `tools/make_test_print.py` 生成测试 raw。

### 3.2 生成测试打印文件

生成 4 行低密度测试数据：

```powershell
python tools/make_test_print.py --pattern low_density_black --lines 4 -o test.bin --summary
```

其他可用图案：

```powershell
python tools/make_test_print.py --pattern blank --lines 4 -o blank.bin
python tools/make_test_print.py --pattern vertical_lines --lines 4 -o lines.bin
python tools/make_test_print.py --pattern checker --lines 4 -o checker.bin
```

真实打印第一次建议只用 `low_density_black`，并且只用 `1..4` 行。

### 3.3 上传文件

```powershell
python tools/api_client.py --base http://<ip> upload test.bin
```

上传工具会自动执行三步：

1. `POST /api/v1/print/files?size=<bytes>&crc32=<crc>` 创建上传会话。
2. `PUT /api/v1/print/files/{file_id}/chunks/{index}` 分片上传 raw 数据。
3. `POST /api/v1/print/files/{file_id}/complete` 校验总大小和 CRC32。

成功后响应里会有：

```json
{
  "file": {
    "file_id": 1,
    "state": "COMPLETE"
  }
}
```

记住这个 `file_id`，启动打印时要用。

## 4. 如何开始打印

### 4.1 启动 mock 打印

如果只启用 `MP_ENABLE_WIFI=1`，没有启用真实热敏头和真实步进电机宏，那么下面命令只会跑软件打印流程，不会真实加热或走纸：

```powershell
python tools/api_client.py --base http://<ip> start <file_id>
```

例如：

```powershell
python tools/api_client.py --base http://192.168.1.134 start 1
```

查询当前打印状态：

```powershell
python tools/api_client.py --base http://<ip> status
python tools/api_client.py --base http://<ip> cancel
```

`start` 内部对应：

- `POST /api/v1/print/jobs?file_id=<id>&copies=1`
- 固件从上传完成的 raw 文件中逐行取 48 字节。
- 每行进入 `PrintSpooler`。
- `PrintEngineTask` 消费行数据。
- 默认 mock 驱动只验证流程，不输出危险硬件动作。

### 4.2 启动真实打印前必须完成的前置验证

真实打印不是上传后直接点 start。必须先完成这些阶段：

1. `safe-off` 后确认 `G_VH=LOW`、`STB1~STB6=LOW`、`G_NSLEEP=LOW`、`AIN/BIN=LOW`。
2. `MP_ENABLE_HW_THERMAL_HEAD=1` 下做 `head-shift-test`，只看 DI/CLK/LAT，VH 必须保持关闭。
3. VH 关闭条件下做 `head-stb-test`，逐组确认 STB 短脉冲。
4. `MP_ENABLE_HW_STEPPER=1` 下做 `motor-test --steps 1`，确认方向和 release/sleep。
5. 确认电池电压正常，低电压时不要打印。
6. 只用低密度、短行数 raw 文件做首次真实打印。

真实硬件宏示例：

```powershell
$env:PLATFORMIO_BUILD_FLAGS='-DMP_ENABLE_WIFI=1 -DMP_ENABLE_HW_THERMAL_HEAD=1 -DMP_ENABLE_HW_STEPPER=1'
python -m platformio run -t upload
Remove-Item Env:PLATFORMIO_BUILD_FLAGS
```

如果还要启用真实传感器骨架：

```powershell
$env:PLATFORMIO_BUILD_FLAGS='-DMP_ENABLE_WIFI=1 -DMP_ENABLE_HW_SENSORS=1 -DMP_ENABLE_HW_THERMAL_HEAD=1 -DMP_ENABLE_HW_STEPPER=1'
python -m platformio run -t upload
Remove-Item Env:PLATFORMIO_BUILD_FLAGS
```

注意：`MP_ENABLE_HW_SENSORS=1` 当前只真实读取 `EQD` 和 `G_NFAULT`；`PAPER_N`、`TM1`、`BAT_STAT` 仍需实测后完善。

## 5. 常用 API 命令

| 操作 | 命令 |
|---|---|
| 查看设备信息 | `python tools/api_client.py --base http://<ip> info` |
| 查看状态 | `python tools/api_client.py --base http://<ip> status` |
| 安全关闭危险输出 | `python tools/api_client.py --base http://<ip> safe-off` |
| 查看传感器 | `python tools/api_client.py --base http://<ip> sensors` |
| 查看电池 | `python tools/api_client.py --base http://<ip> battery` |
| 上传 raw 文件 | `python tools/api_client.py --base http://<ip> upload test.bin` |
| 从文件启动打印 | `python tools/api_client.py --base http://<ip> start <file_id>` |
| 取消当前打印 | `python tools/api_client.py --base http://<ip> cancel` |
| 走纸 | `python tools/api_client.py --base http://<ip> feed --steps 10` |
| 电机低速测试 | `python tools/api_client.py --base http://<ip> motor-test --steps 1` |
| 打印头 shift/latch 测试 | `python tools/api_client.py --base http://<ip> head-shift-test --file test.bin` |
| 单组 STB 测试 | `python tools/api_client.py --base http://<ip> head-stb-test --group 0 --pulse-us 5` |

## 6. 串口现在怎么用

串口当前只建议用于：

- 烧录固件。
- 查看启动日志。
- 临时底层调试。

如果调试时 USB 给开发板供电、底板电池给外设供电，应保持：

- 断开底板给开发板的正电源跳帽。
- ESP32 和底板 GND 必须共地。
- 不要在 USB 插着时重新合上供电跳帽。
- 先执行或确认 `safe-off`，再进入任何外设动作测试。

正式打印文件传输和启动打印仍然使用 WiFi API，不使用串口 `PRINT_LINE/PRINT_START` 作为正式入口。

### 6.1 启动日志怎么看

正常上电时，前几行 `ESP-ROM:esp32s3-20210327`、`rst:0x1 (POWERON)`、`boot:0x28 (SPI_FAST_FLASH_BOOT)` 是 ESP32-S3 ROM 和 bootloader 日志，表示芯片从 Flash 启动。

如果看到：

```text
Guru Meditation Error
Stack canary watchpoint triggered (task_log)
Rebooting...
```

这不是正常启动，而是 FreeRTOS 任务栈溢出后触发保护并重启。当前已把各任务 `xTaskCreate()` 的栈大小按 ESP32 Arduino 的“字节”单位重新调整，避免 `task_log` 因 1024 bytes 栈过小而崩溃。

如果看到：

```text
Preferences.cpp: begin(): nvs_open failed: NOT_FOUND
```

表示第一次启动时 NVS 参数命名空间还不存在。当前 NVS 初始化已改为先用读写模式创建命名空间，后续不应再因首次启动打印这条错误。

## 7. 最小完整流程

下面是从生成文件到启动 mock 打印的最小流程：

```powershell
# 1. 烧录 WiFi API 固件
$env:PLATFORMIO_BUILD_FLAGS='-DMP_ENABLE_WIFI=1'
python -m platformio run -t upload
Remove-Item Env:PLATFORMIO_BUILD_FLAGS

# 2. 检查设备
python tools/api_client.py --base http://<ip> info
python tools/api_client.py --base http://<ip> safe-off

# 3. 生成 raw 测试文件
python tools/make_test_print.py --pattern low_density_black --lines 4 -o test.bin --summary

# 4. 上传文件，记录响应里的 file_id
python tools/api_client.py --base http://<ip> upload test.bin

# 5. 启动打印
python tools/api_client.py --base http://<ip> start <file_id>

# 6. 查询或取消
python tools/api_client.py --base http://<ip> status
python tools/api_client.py --base http://<ip> cancel
```

真实打印时，把第 5 步放到所有硬件分阶段验收之后再执行。
