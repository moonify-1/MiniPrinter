## 1. 仍需补充的信息

| 编号 | 需要补充的信息 | 当前处理方式 | 补充后用途 |
|---|---|---|---|
| R01 | `PAPER_N` 有纸/无纸时的实际电平 | 第一阶段先按“始终有纸”处理，并在代码中写 TODO | 后续接入真实缺纸检测和缺纸停机保护 |
| R02 | `TM1` 室温和升温后的 ADC 原始值 | 第一阶段先保留 mock/保守温度，并写 TODO | 后续校准 30k/B3950 NTC 温度换算 |
| R03 | IP2326 `BAT_STAT` 的准确状态语义 | 后续查 `项目背景/电源管理芯片IP2326.pdf`，不要猜测 | 后续把充电中、充满、异常状态映射到 `ChargeStatus` |
| R04 | 电机正向走纸相序 | 先查机芯/DRV8833 数据手册；不明确时低速实测 | 后续修正 `MOTOR_TEST` 和打印走纸方向 |
| R05 | WiFi 测试 API 的具体协议形式 | 当前只确认设备局域网 IP 为 `192.168.1.134` | 后续替代串口调试，执行 `STATUS/SENSOR_TEST/SAFE_OFF` 等测试 |

## 2. 仍需要你亲自完成的任务

| 阶段 | 任务 | 工具 | 完成标准 |
|---|---|---|---|
| P01 | 测 `PAPER_N` 有纸/无纸电平 | 万用表或逻辑分析仪 | 记录有纸、无纸两个状态的电平 |
| P02 | 提供 `TM1` ADC 原始值 | 串口/WiFi 测试 API 或临时调试输出 | 记录室温值，并观察轻微升温后的变化方向 |
| P03 | 查明 `BAT_STAT` 状态语义 | IP2326 数据手册、万用表或逻辑分析仪 | 能区分未充电、充电中、充满、异常中的至少主要状态 |
| P04 | 低速小步数电机方向测试 | 实物机芯 | 确认正向走纸相序，测试后电机能 release |
| P05 | 空载验证 shift/latch/STB 波形 | 示波器或逻辑分析仪 | DI/CLK/LAT/STB 时序符合预期，且 VH 保持关闭 |
| P06 | 首次真实加热打印 | 示波器、温度观察、短测试数据 | 只允许低黑点密度、短行数，出现异常立即断电 |

## 3. 当前不应继续追问的事项

- MCU 型号已确认：`ESP32-S3-WROOM-1-N16R8`。
- DevKit 插接方向已确认：Wi-Fi 天线在上时左右与原理图一致。
- 网络到 GPIO 连通性已由用户复核。
- GPIO35~40 已确认可作为普通 GPIO 输出。
- GPIO38 不再作为 RGB LED 冲突风险；板载 LED 真实使用 GPIO47，且本项目不使用该 LED。
- `G_VH` 控制方向已确认：高通低断。
- `EQD` 分压按 220k/100k 处理。
- 电池方案已确认：两个 18650 串联的 2S 方案。
- H3 不作为固件运行期控制对象。
