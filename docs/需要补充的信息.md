本文记录 Codex 无法从现有仓库资料中完全确定的问题，以及必须由你基于真实硬件、万用表、示波器、逻辑分析仪或实际测试亲自完成的事项。

| H03 | 用万用表确认每个网络到 ESP32 GPIO 的连通性 | 目前 GPIO 表是根据原理图和映射图推断，未实物复核 | 待填写 |
| H04 | `GPIO38` 是否被板载 RGB LED 占用且是否影响 `STB3` | GPIO38 在映射图中标注 RGB_LED，可能影响热敏头 STB3 | 待填写 |

| H07 | `PAPER_N` 有纸/无纸时的实际电平 | 代码计划按低有效处理，必须用纸张插入/移除实测确认 | 待填写 |

| H14 | H3 跳线或供电选择的实际接法 | 原理图中 H3 影响 MCU 供电路径，固件文档需要知道真实配置 | 待填写 |
| H15 | 串口连接方式和真实端口号 | 主机工具 `send_frame.py` 需要实际串口号和波特率验证 | 待填写 |

## 2. Codex 当前不确定的问题

- `docs/硬件匹配分析.md` 的 GPIO 映射是根据排针位置推断的，还没有实物连通性验证。
- `GPIO35~40` 用作 `STB1~STB6` 是否会与当前 DevKit 板载功能、JTAG 或 SPI 复用冲突，还不能确定。
- `GPIO38` 同时标注 `RGB_LED`，实际是否影响 `STB3` 需要看当前板卡原理图和实测。
- `BAT_STAT` 的电平语义还没有从 IP2326 资料和实测中完全闭环。
- `EQD` 分压比按原理图估算为 3.2，但 ADC 校准、电阻误差和满电电压都需要实测。
- `TM1` 使用 30k/B3950 NTC 的方向基本明确，但 ADC 读数到温度的公式仍需要结合实际上拉电阻、ADC 衰减和校准确认。
- 一行打印后应走 2 步的方向很明确，但真实走纸方向、噪声、丢步和速度上限需要机芯实测。
- 热敏头脉宽模型目前只能保守估算，真实打印浓度和温升必须低密度、短行数逐步验证。

## 3. 必须你亲自完成的任务

| 阶段 | 任务 | 工具 | 完成标准 |
|---|---|---|---|
| P01 | 复核 DevKit 插接方向 | 目视、照片、原理图 | `Left/Right` 与实物方向确认无误 |
| P02 | 逐项测量 GPIO 连通性 | 万用表蜂鸣档 | 每个网络都能对应到唯一 ESP32 GPIO |
| P03 | 上电但不接打印头，测安全态 | 万用表 | `VH_OUT` 默认关闭，STB 默认 inactive，电机不动作 |
| P04 | 运行 `SAFE_OFF` 后测危险输出 | 万用表 | VH、STB、电机控制脚全部处于安全态 |
| P05 | 测 `PAPER_N` 有纸/无纸电平 | 万用表或逻辑分析仪 | 记录有纸、无纸两个状态的电平 |
| P06 | 测 `BAT_STAT` 各状态电平 | 万用表或逻辑分析仪 | 记录未充电、充电中、充满、异常状态 |
| P07 | 测 `EQD` 分压输出 | 万用表 | 记录电池总压和 ADC 脚电压，确认不超过 3.3V |
| P08 | 测 `TM1` ADC 输入 | 万用表或串口输出 | 记录室温下电压/ADC，并用手温或轻微加热观察变化方向 |
| P09 | 空载验证 shift/latch/STB 波形 | 示波器或逻辑分析仪 | DI/CLK/LAT/STB 时序符合预期，且 VH 保持关闭 |
| P10 | 低速小步数电机测试 | 实物机芯 | 电机方向正确、无明显卡顿、测试后 release |
| P11 | VH 关闭条件下做 STB 测试 | 示波器 | STB 脉冲宽度符合命令参数，VH 不打开 |
| P12 | 首次真实加热打印 | 示波器、温度观察、短测试数据 | 只允许低黑点密度、短行数，出现异常立即断电 |

## 4. 建议你补充到仓库的资料

- DevKit 实物正反面照片，能看清板卡版本、模组型号和排针方向。
- 自制 PCB 上 DevKit 插接后的照片。
- 万用表连通性表格，建议直接补到 `docs/pin_map.md`。
- `PAPER_N`、`BAT_STAT`、`EQD`、`TM1` 的实测表格。
- 首次 `SENSOR_TEST` 的串口输出。
- 空载 GPIO 波形截图。
- 电机低速测试结果和走纸方向记录。
- 首次低密度打印结果照片。
