# MiniPrinter 热敏头热模型

更新时间：2026-04-26 21:53:40

## 1. 目标

本文件记录当前固件的第一阶段热模型。它的目标不是追求最终打印浓度，而是让真实打印前的脉宽计算具备可解释、安全、可校准的基线。

## 2. 当前规格基线

| 项目 | 当前值 | 来源和说明 |
|---|---:|---|
| 打印宽度 | `384 dots` | JX-2R-01 机芯规格，固件按 48 字节/行处理。 |
| STB 分组 | `6` | 每组 64 dots。 |
| 最大同时加热点 | `64 dots` | 规格书限制，固件继续硬限制单组不超过 64 dots。 |
| 参考 VH | `7200 mV` | 规格书推荐工作点，实际由 2S 电池经 `G_VH` 高边开关输出。 |
| 典型 Ton | `490 us` | 25C、VH=7.2V、64 dots 同时加热的典型参考。 |
| 单组硬上限 | `500 us` | 3000us 行加热预算 / 6 组，防止整行加热阶段过长。 |
| 全局硬上限 | `800 us` | `SAFETY_DEFAULT_HEAT_PULSE_MAX_US`，仍作为绝对保护线。 |

## 3. 计算模型

当前代码在 `src/services/thermal_safety_service.cpp` 中按以下顺序计算：

1. 先执行安全检查：缺纸、传感器无效、电机故障、过温、低电压任一失败都会拒绝加热。
2. 再检查单组黑点数：超过 `64 dots` 直接返回错误。
3. 以 `heatPulseStartUs` 作为低密度起始脉宽，默认 `400us`。
4. 根据当前 STB 组黑点数，把脉宽从起始值平滑推向 `490us` 典型满载参考。
5. 根据头温补偿：高于 25C 时减少脉宽，低于 25C 时只允许小幅增加。
6. 根据电压补偿：按电阻负载近似关系 `time ~= (7200 / battery_mv)^2` 缩放。
7. 最后同时受 `heatPulseMaxUs`、`800us` 全局硬上限、`500us` 单组行预算限制。

## 4. 仍需实测

- `TM1` 目前仍未完成真实 ADC 到温度的校准，真实过温保护不能宣称完成。
- `battery_mv` 由 `EQD` 按 220k/100k 分压估算，ADC 校准和 VH_OUT 实际压降仍待实测。
- 打印浓度、纸张差异、MOS 压降、FPC 线阻和连续行温升都还没有进入模型。
- 首次真实加热只能使用低密度、短行数、可立即断电的测试图案。
